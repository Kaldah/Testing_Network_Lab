cleanup_stale_rules(ttl_seconds=7200, heartbeat_dir='/run/stormshadow', dry_run=False)[source]¶

Remove Stormshadow-tagged rules from our dedicated chains if they appear stale. A rule is considered stale if its embedded timestamp is older than ttl_seconds and there’s no recent heartbeat file for its SUID (mtime > now - ttl_seconds/2).

First cleans up stale heartbeat files, then removes corresponding rules. Returns number of rules removed.

Parameters:

  • ttl_seconds (int)

  • heartbeat_dir (str)

  • dry_run (bool)

ensure_nfqueue_rule_using_ipset(queue_num, suid, anchor_chain='INPUT', set_timeout=7200, dry_run=False)[source]¶

Better approach: use ipset to control which ports are sent to NFQUEUE, with per-port TTL.

  • Create ipset set name “stormshadow_ports_{suid}” (bitmap:port, timeout).

  • Insert a single iptables rule in STORMSHADOW chain matching

    “-m set –match-set <set> dst” and sending to NFQUEUE.

  • Entries (ports) in the set expire automatically.

Returns the set name if successful, else None.

Parameters:

  • queue_num (int)

  • suid (str)

  • anchor_chain (str)

  • set_timeout (int)

  • dry_run (bool)

Return type:

str | None
