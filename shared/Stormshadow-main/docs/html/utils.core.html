utils.core.command_runner module¶

run_command(argv, *, cwd=None, env=None, want_sudo=False, sudo_non_interactive=True, sudo_preserve_env=False, capture_output=True, check=True, text=True, dry_run=False)[source]¶

Run a command as a list of arguments and return a CompletedProcess result.

Features:

  • No shell: arguments are passed directly to the OS, avoiding shell injection risks.

  • Automatic sudo support: if want_sudo is True and the current user is not root, the

    command is prefixed with sudo (if available). If sudo_non_interactive is True, “-n” is added to fail fast instead of waiting for a password prompt. If sudo_non_interactive is False, interactive prompts are allowed. If sudo_preserve_env is True, “-E” is added to preserve the current environment variables under sudo.

  • Environment control: override cwd (working directory) and env (environment variables).

  • Output capture: by default, both stdout and stderr are captured (capture_output=True), and

    the output is decoded to text strings (text=True).

  • Error handling: if check=True, raise subprocess.CalledProcessError on non-zero exit; otherwise

    return the CompletedProcess object without raising.

Parameters:

  • argv (List[str]) – List of program arguments, for example [“ls”, “-l”].

  • cwd (str | None) – Optional working directory to run the command in.

  • env (Dict[str, str] | None) – Optional dict of environment variables to pass to the command.

  • want_sudo (bool) – Whether to prefix the command with sudo when not root.

  • sudo_non_interactive (bool) – If True, pass -n to sudo to fail fast if a password is required. If False, allow interactive password prompts.

  • sudo_preserve_env (bool) – If True, pass -E to sudo to keep the current environment.

  • capture_output (bool) – If True, capture stdout and stderr into the CompletedProcess object.

  • check (bool) – If True, raise an exception if the command returns a non-zero exit code.

  • text (bool) – If True, decode stdout/stderr to str; if False, return bytes.

  • dry_run (bool)

Returns:

subprocess.CompletedProcess – The result object with attributes args, returncode, stdout (if capture_output=True), and stderr (if capture_output=True).

Raises:

  • subprocess.CalledProcessError – If check=True and the command fails.

  • RuntimeError – If want_sudo=True but sudo is requested/required and not available.

Return type:

CompletedProcess[str]

class ConsoleWindow[source]¶

Bases: object

Tk console that mirrors a running process.

Exactly one of:

  • process: an existing Popen created with stdin=PIPE, stdout=PIPE (stderr optional)

  • io: a TerminalIO backend (e.g., PTY-backed on Unix)

Closing the window does NOT kill the process. The window auto-closes when the process exits.
